================================================================================
  HOW TO ADD VARIABLES
================================================================================

1. Go to Railway Dashboard
2. Click on your project
3. Click "Variables" tab
4. Click "New Variable"
5. Paste Name and Value from above
6. Click "Save"
7. Repeat for all variables


================================================================================
  IMPORTANT: Railway Keeps Switching Back to Nixpacks!
================================================================================

If Railway keeps switching back to Nixpacks automatically:

SOLUTION 1: Delete and Recreate Service
--------------------------------------------
1. Go to Settings
2. Scroll to bottom
3. Click "Delete Service"
4. Create NEW service
5. Connect to GitHub repo
6. Railway will auto-detect Dockerfile


SOLUTION 2: Remove Nixpacks Files from GitHub
--------------------------------------------
Make sure these files are NOT in your GitHub repo:
- nixpacks.toml (should be deleted)
- Any .nix files

Our .gitignore should already exclude these.


SOLUTION 3: Add .railway Directory
--------------------------------------------
Create a folder called .railway in your project root
Create a file inside called .ignore
Leave it empty

This sometimes forces Docker detection.


SOLUTION 4: Contact Railway Support
--------------------------------------------
If nothing works, Railway support can help:
https://railway.com/help

They can manually switch your build provider.


================================================================================
  AFTER SETTING VARIABLES
================================================================================

1. Make sure Build Provider = Dockerfile
2. Add all environment variables above
3. Click "Deploy"
4. Wait 3-5 minutes
5. Should succeed!


================================================================================
  TROUBLESHOOTING
================================================================================

Problem: Railway switches back to Nixpacks
Solution: Delete service and recreate, or contact support

Problem: Build fails with "npm ci" error
Solution: Make sure Build Provider is set to Dockerfile, not Nixpacks

Problem: Variables not showing up
Solution: Click "Refresh" or reload the page

Problem: Don't know your Railway URL
Solution: Check Deployments tab after successful deploy
